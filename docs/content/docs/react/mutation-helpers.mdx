---
title: Mutation helpers
description: Every callback returned by the hook, and the action it dispatches.
---

The hook returns eight mutation callbacks alongside `query`. Each is a stable
(`useCallback`) function that **normalizes** any node input for you and dispatches the
corresponding [reducer action](/docs/core/reducer). Targets are addressed by node `id`.

## Adding

### addCondition

```ts
addCondition(groupId: string, node: ConditionNodeInput): void
```

Append a condition to `groupId`. Dispatches `APPEND_NODE`.

```ts
qb.addCondition(qb.query.id, {
  type: "condition", field: "age", operator: "gt", value: 30,
});
```

### addGroup

```ts
addGroup(groupId: string, operator: GroupOperatorId, children?: QueryNodeInput[]): void
```

Append a nested group (optionally pre-filled with `children`) to `groupId`. Dispatches
`APPEND_NODE`. `children` defaults to `[]`.

```ts
qb.addGroup(qb.query.id, "or", [
  { type: "condition", field: "status", operator: "eq", value: "active" },
]);
```

## Inserting at an index

### insertCondition

```ts
insertCondition(groupId: string, index: number, node: ConditionNodeInput): void
```

Insert a condition at `index` in `groupId`. Dispatches `INSERT_NODE` (index is clamped).

### insertGroup

```ts
insertGroup(groupId: string, index: number, operator: GroupOperatorId, children?: QueryNodeInput[]): void
```

Insert a nested group at `index`. Dispatches `INSERT_NODE`.

## Removing & moving

### removeNode

```ts
removeNode(nodeId: string): void
```

Remove `nodeId`. Dispatches `REMOVE_NODE`. Removing the root is a no-op.

### moveNode

```ts
moveNode(nodeId: string, targetGroupId: string, index?: number): void
```

Move `nodeId` into `targetGroupId` (appends if `index` is omitted). Dispatches `MOVE_NODE`,
with all of [`moveNode`'s guards](/docs/core/tree-helpers#movenode). See the
[drag & drop guide](/docs/guides/drag-and-drop) for the same-group index adjustment.

## Updating

### updateCondition

```ts
updateCondition(nodeId: string, patch: Partial<ConditionNodeInput>): void
```

Patch a condition. **Before dispatching**, the hook runs
[`buildConditionPatch`](/docs/core/build-condition-patch) — so if your `patch` changes the
`field` to one that doesn't support the current operator, the operator resets to the new
field's default and the value clears. Then it dispatches `UPDATE_NODE`.

```ts
qb.updateCondition(id, { value: "Bob" });   // simple value change
qb.updateCondition(id, { field: "age" });   // operator/value reset if incompatible
```

### updateGroupOperator

```ts
updateGroupOperator(groupId: string, operator: GroupOperatorId): void
```

Change a group's logical operator (e.g. `and` → `or`). Dispatches `UPDATE_GROUP_OPERATOR`.
The first argument is the **id of the group node**; the second is the new operator id.

<Callout>
All inputs are normalized inside the helper, so you never pass ids for new nodes — write
`{ type: "condition", field, operator, value }` and the hook assigns the id.
</Callout>
