`@grnrb/react-query-builder` is a thin React layer over the core. It adds **one hook** —
[`useQueryBuilder`](/docs/react/use-query-builder) — and the types describing its options
and return value. Everything else you import from it (`defineSchema`, `reducer`, the tree
helpers, all the core types) is **re-exported from the core**, so a React app only needs
this one package.

```tsx
import {
  defineSchema,
  useQueryBuilder,
  // ...plus everything from @grnrb/query-builder-core
} from "@grnrb/react-query-builder";
```

## What the hook does for you

The hook wires the [`reducer`](/docs/core/reducer) and [tree helpers](/docs/core/tree-helpers)
to React state and hands back ergonomic, **stable** mutation callbacks:

- It manages state with `useReducer` (uncontrolled) or defers to your state
  (controlled) — see [Controlled vs uncontrolled](/docs/react/controlled-vs-uncontrolled).
- It **normalizes** the nodes you pass in (assigns ids), so you write inputs without ids.
- Its `updateCondition` runs [`buildConditionPatch`](/docs/core/build-condition-patch)
  automatically, so changing a field resets an incompatible operator/value.

## Requirements

`react` (>= 18) is a peer dependency. The hook uses only `useReducer` and `useCallback`, so
it works in any React 18+ app (and in React 19, used by the
[playground](/docs/getting-started/playground)).
