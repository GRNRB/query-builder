```ts
function useQueryBuilder<const TSchema extends QuerySchema>(
  schema: TSchema,
  options: UseQueryBuilderOptions<TSchema>,
): UseQueryBuilderReturn<TSchema>
```

The hook takes your [schema](/docs/concepts/schema) and an options object, and returns the
current `query` plus a set of [mutation helpers](/docs/react/mutation-helpers).

```tsx
const qb = useQueryBuilder(schema, {
  initialQuery: {
    type: "group",
    operator: "and",
    children: [
      { type: "condition", field: "name", operator: "eq", value: "Alice" },
    ],
  },
});

qb.query;            // the current, normalized tree
qb.addCondition(/*…*/);
```

## Options

The options are a **union**: either uncontrolled (`initialQuery`) or controlled
(`query` + `onChange`). Both accept an optional `generateId`. See
[Controlled vs uncontrolled](/docs/react/controlled-vs-uncontrolled) for when to use each.

<AutoTypeTable path="../packages/react-query-builder/src/types.ts" name="UseQueryBuilderOptions" />

- **`initialQuery`** — uncontrolled. The hook owns state via `useReducer`; pass a
  `QueryNodeInput` (ids optional, filled in by [`normalizeTree`](/docs/core/tree-helpers#normalizetree)).
- **`query` + `onChange`** — controlled. You own the state; the hook computes the next tree
  and calls `onChange` with it.
- **`generateId`** — override id generation (defaults to
  [`defaultGenerateId`](/docs/core/tree-helpers#defaultgenerateid)). See
  [Custom id generation](/docs/guides/custom-id-generation).

## Return value

<AutoTypeTable path="../packages/react-query-builder/src/types.ts" name="UseQueryBuilderReturn" />

`query` is the current tree; the rest are **stable callbacks** (memoized with
`useCallback`) — safe to pass to memoized children. Each is documented in
[Mutation helpers](/docs/react/mutation-helpers).
