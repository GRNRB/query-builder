Everything is exported from `@grnrb/query-builder-core` and **re-exported** from
`@grnrb/react-query-builder` (which adds the [`useQueryBuilder`](/docs/framework/react)
hook) — so a React app imports only this one package. For how to *use* these to change a
tree, see the [React guide](/docs/framework/react).

## Functions

### defineSchema

```ts
function defineSchema<const TSchema extends QuerySchema>(schema: TSchema): TSchema
```

An identity function with a `const` type parameter: it returns the schema unchanged but
**preserves its literal types** (powering operator-per-field inference) and, in
non-production builds, **warns** about a duplicate field id, a duplicate condition operator
id, or a `defaultOperator` not in `supportedOperators`. See
[Concepts](/docs/framework/react/concepts).

### normalizeTree

```ts
function normalizeTree(node: QueryNodeInput, generateId?: () => string): QueryNode
```

Recursively assigns an `id` to any node missing one. Pass a custom `generateId` to control
id generation (defaults to `defaultGenerateId`, i.e. `crypto.randomUUID()`). See
[Custom id generation](/docs/framework/react#custom-id-generation).

### reducer

```ts
function reducer<TSchema>(state: QueryNode<TSchema>, action: Action<TSchema>): QueryNode<TSchema>
```

Takes the current tree and an action, returns the next tree — **two arguments, no schema
parameter**. Each action maps to a tree helper, so it inherits the same immutability and
guards (an action that changes nothing returns the same reference). The hook's helpers
dispatch these for you. The `Action` union:

| `type`                  | Fields                                       |
| ----------------------- | -------------------------------------------- |
| `APPEND_NODE`           | `groupId`, `node`                            |
| `INSERT_NODE`           | `groupId`, `index`, `node`                   |
| `MOVE_NODE`             | `nodeId`, `targetGroupId`, `index?`          |
| `REMOVE_NODE`           | `nodeId`                                     |
| `UPDATE_NODE`           | `nodeId`, `patch` (condition only)           |
| `UPDATE_GROUP_OPERATOR` | `groupId`, `operator`                        |

### buildConditionPatch

```ts
function buildConditionPatch<TSchema>(
  schema: TSchema,
  current: QueryNode<TSchema>,
  patch: Partial<ConditionNodeInput<TSchema>>,
): Partial<ConditionNodeInput<TSchema>>
```

Resolves a condition update: if the `patch` changes the `field` to one that doesn't support
the current operator, it resets the operator to the new field's `defaultOperator` (or first
supported) and clears `value`; otherwise it returns the patch unchanged. The
`updateCondition` helper runs this automatically.

### Tree helpers

Immutable building blocks — each returns a new tree, or the *same reference* when nothing
changed.

| Function | Signature |
| -------- | --------- |
| `findNode` | `(root, id) => QueryNode \| undefined` — depth-first lookup |
| `updateNode` | `(root, id, updater) => QueryNode` — apply `updater` to one node |
| `removeNode` | `(root, id) => QueryNode` — remove a node (never the root) |
| `appendToGroup` | `(root, groupId, node) => QueryNode` — append as last child |
| `insertIntoGroup` | `(root, groupId, index, node) => QueryNode` — insert (index clamped) |
| `moveNode` | `(root, nodeId, targetGroupId, index?) => QueryNode` — move, with guards |
| `defaultGenerateId` | `() => string` — `crypto.randomUUID()` |

## The hook

### useQueryBuilder

```ts
function useQueryBuilder<TSchema>(
  schema: TSchema,
  options: UseQueryBuilderOptions<TSchema>,
): UseQueryBuilderReturn<TSchema>
```

Holds (or wraps) the query tree and returns it alongside stable, memoized mutation helpers.
Pass `initialQuery` for uncontrolled state, or `query` + `onChange` for controlled — see
[Controlled vs uncontrolled](/docs/framework/react#controlled-vs-uncontrolled).

| -------------------------------- | ------- |
| `Arity`                          | `"none" \| "one" \| "two" \| "many"`. |
| `ValueForArity<TArity, TValue>`  | Maps an arity to its [value shape](/docs/framework/react/concepts#operators-and-arity). |
