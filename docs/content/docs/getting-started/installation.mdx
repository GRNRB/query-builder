---

Pick the package that matches your stack. Both ship **dual ESM/CJS builds with full
TypeScript types**.

## React

If you're building a React UI, install the React package. It **re-exports the entire
core**, so you don't need to install the core separately.

```sh
npm install @grnrb/react-query-builder
# or: pnpm add @grnrb/react-query-builder
# or: yarn add @grnrb/react-query-builder
```

`react` (>= 18) is a **peer dependency** — it is not bundled, so your app's React copy is
used.

```ts
import { defineSchema, useQueryBuilder } from "@grnrb/react-query-builder";
```

## Core only

For non-React usage (Node, Vue, Svelte, a server, tests…) install just the core:

```sh
npm install @grnrb/query-builder-core
# or: pnpm add @grnrb/query-builder-core
```

```ts
import { defineSchema, normalizeTree, reducer } from "@grnrb/query-builder-core";
```

<Callout title="Which one do I import from?">
The React package re-exports everything in the core (`defineSchema`, `reducer`, the tree
helpers, and all types). In a React app, import everything from
`@grnrb/react-query-builder` — there's no need to import from the core directly.
</Callout>
