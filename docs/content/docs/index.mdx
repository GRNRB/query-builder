**query-builder** is a small toolkit for building **structured query / filter trees** — the
kind of nested *condition* and *group* UI you see behind "Add filter" buttons in dashboards,
audience builders, and search interfaces.

It ships as two packages:

## What it gives you

- **A typed schema** — declare your fields, the operators they support, and how groups
  combine (`and` / `or`). The schema *drives the types*: TypeScript knows which operators
  a field allows and what shape each value takes.
- **An immutable tree** — a query is plain JSON: a root `group` whose children are
  `condition` leaves or nested `group`s. Every helper returns a new tree (with structural
  sharing) and never mutates in place.
- **Pure transforms** — add, insert, move, remove, and update nodes through small helpers
  or a single `reducer`. No classes, no hidden state.

## A taste

```ts
import { defineSchema, normalizeTree } from "@grnrb/query-builder-core";

const schema = defineSchema({
  fields: [
    { id: "name", label: "Name", supportedOperators: ["eq", "contains"] },
    { id: "age", label: "Age", supportedOperators: ["eq", "gt", "lt"] },
  ],
  conditionOperators: [
    { id: "eq", arity: "one", label: "equals" },
    { id: "contains", arity: "one", label: "contains" },
    { id: "gt", arity: "one", label: "greater than" },
    { id: "lt", arity: "one", label: "less than" },
  ],
  groupOperators: [
    { id: "and", arity: "many", label: "AND" },
    { id: "or", arity: "many", label: "OR" },
  ],
});

// A query is just data. `normalizeTree` fills in any missing ids.
const query = normalizeTree({
  type: "group",
  operator: "and",
  children: [
    { type: "condition", field: "name", operator: "eq", value: "Alice" },
    { type: "condition", field: "age", operator: "gt", value: 18 },
  ],
});
```
