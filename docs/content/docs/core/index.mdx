`@grnrb/query-builder-core` is the whole engine: it defines the [schema](/docs/core/schema),
the immutable [query tree](/docs/core/query-tree), the [reducer](/docs/core/reducer), and the
[tree helpers](/docs/core/tree-helpers). It has **no dependencies and no framework** — use it
from Node, a server, tests, or any UI layer.

Everything documented here is exported from the core and **re-exported** from
`@grnrb/react-query-builder` (which adds the [`useQueryBuilder`](/docs/frameworks/react/api)
hook). A React app installs only the React package; a non-React app installs the core
directly:

```sh
npm install @grnrb/query-builder-core
```

## How the pieces fit

A **schema** describes what queries are allowed. A **query tree** is one such query — plain,
immutable JSON whose root is always a group. You change a tree by feeding it (with an
[action](/docs/core/reducer)) to the `reducer`, which returns the next tree; each action maps
to an equivalent [tree helper](/docs/core/tree-helpers) you can also call directly. Every
operation is immutable and guarded — it returns a new tree, or the *same reference* when
nothing changed.
