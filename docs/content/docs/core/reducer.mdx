---
title: reducer
description: The action-driven entry point ‚Äî (state, action) => state.
---

```ts
function reducer<TSchema>(
  state: QueryNode<TSchema>,
  action: Action<TSchema>,
): QueryNode<TSchema>
```

`reducer` takes the **current tree and an action**, and returns the next tree. It takes
exactly two arguments ‚Äî there is no separate `schema` parameter ‚Äî and is a drop-in for
React's `useReducer` or any store.

```ts
import { reducer } from "@grnrb/query-builder-core";

let tree = reducer(tree, {
  type: "UPDATE_NODE",
  nodeId: someId,
  patch: { value: "Bob" },
});
```

Each action maps to one [tree helper](/docs/core/tree-helpers), so the reducer inherits
the same immutability and guards ‚Äî an action that changes nothing returns the same `state`
reference.

## Actions

The `Action` union has six members. Action `type`s are written in **`SCREAMING_SNAKE_CASE`**.

| `type`                   | Fields                                              | Effect |
| ------------------------ | --------------------------------------------------- | ------ |
| `APPEND_NODE`            | `groupId: string`, `node: QueryNode`                | Append `node` as the last child of `groupId`. |
| `INSERT_NODE`            | `groupId: string`, `index: number`, `node: QueryNode` | Insert `node` at `index` (clamped) in `groupId`. |
| `MOVE_NODE`              | `nodeId: string`, `targetGroupId: string`, `index?: number` | Move `nodeId` into `targetGroupId` (appends if no `index`). |
| `REMOVE_NODE`            | `nodeId: string`                                    | Remove `nodeId` (never the root). |
| `UPDATE_NODE`            | `nodeId: string`, `patch: Partial<ConditionNodeInput>` | Shallow-merge `patch` into the **condition** node `nodeId`. |
| `UPDATE_GROUP_OPERATOR`  | `groupId: string`, `operator: string`               | Set the group node `groupId`'s `operator`. |

<Callout type="warn" title="UPDATE_NODE only patches conditions">
`UPDATE_NODE` merges `patch` into the target only when it is a **condition** node; a group
node is returned unchanged. Note also that `UPDATE_NODE` is a *raw* merge ‚Äî unlike the
hook's `updateCondition`, it does **not** reset the operator/value when you change the
field. Use [`buildConditionPatch`](/docs/core/build-condition-patch) (or the
[hook](/docs/react/mutation-helpers#updatecondition)) if you need that behavior.
</Callout>

## Example: a small loop

```ts
import { normalizeTree, reducer } from "@grnrb/query-builder-core";

let tree = normalizeTree({ type: "group", operator: "and", children: [] });

tree = reducer(tree, {
  type: "APPEND_NODE",
  groupId: tree.id,
  node: normalizeTree({ type: "condition", field: "age", operator: "gt", value: 18 }),
});

tree = reducer(tree, { type: "UPDATE_GROUP_OPERATOR", groupId: tree.id, operator: "or" });
```

<Callout>
Nodes passed to `APPEND_NODE` / `INSERT_NODE` / `MOVE_NODE` must already be **normalized**
(have ids). The React [hook](/docs/react/mutation-helpers) normalizes inputs for you; when
calling the reducer directly, wrap new nodes in [`normalizeTree`](/docs/core/tree-helpers#normalizetree).
</Callout>
