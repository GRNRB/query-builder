---
title: Serializing queries
description: Persist and restore a query — it's already plain JSON.
---

A query tree is plain, serializable data: a nested structure of objects, strings, arrays,
and primitive values. There's nothing to "convert" — you can `JSON.stringify` it directly.

## Saving

```ts
localStorage.setItem("filters", JSON.stringify(qb.query));
// or send it to your API as the request body
```

The serialized form is exactly the tree you see in the
[playground's](/docs/getting-started/playground) JSON panel.

## Restoring

A persisted tree **already has ids**, so you can parse and use it as-is:

```ts
const saved = JSON.parse(localStorage.getItem("filters")!) as QueryNode<typeof schema>;

// uncontrolled
const qb = useQueryBuilder(schema, { initialQuery: saved });

// controlled
const [query, setQuery] = useState(() => saved);
```

If instead you're loading a **template authored without ids**, normalize it on the way in:

```ts
import { normalizeTree } from "@grnrb/react-query-builder";

const tree = normalizeTree(templateWithoutIds);
```

(`normalizeTree` only fills in *missing* ids, so it's safe either way.)

## Turning a query into a backend filter

The tree is intentionally backend-agnostic — translating it into SQL, a Mongo filter, an
Elasticsearch query, etc. is a straightforward recursive walk: handle the two `type`s, and
read each operator's [arity](/docs/concepts/conditions-and-arity) to interpret `value`.

```ts
function toPredicate(node: QueryNode<typeof schema>): unknown {
  if (node.type === "group") {
    return { [node.operator]: node.children.map(toPredicate) };
  }
  // node.type === "condition"
  return { field: node.field, op: node.operator, value: (node as any).value };
}
```

<Callout>
Keep your **schema** as the source of truth on both ends. Validate an incoming tree against
the current schema (fields/operators still exist) before trusting persisted queries that
may predate a schema change.
</Callout>
