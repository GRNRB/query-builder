The widget below is a real `useQueryBuilder` instance running in your browser. Add conditions,
change fields and operators, nest sub-groups, and **drag the handle (⠿) to reorder** — the JSON
underneath updates live.

## How it's built

This is a trimmed version of the standalone
[`examples/react/basic`](https://github.com/GRNRB/query-builder/tree/main/examples/react/basic)
app (~600 lines), which imports `@grnrb/react-query-builder` straight from this repository — so
it always exercises the current source. A few things worth copying:

| Piece          | What it does                                                         | Related |
| -------------- | ------------------------------------------------------------------- | ------- |
| `schema`       | Fields, operators (incl. a `none`-arity `is_set`), and `and`/`or`.  | [Schema](/docs/core/schema) |
| `ConditionRow` | Field + operator selects, and a value input **only for `arity: "one"`**. | [Arity](/docs/core/schema#operators-and-arity) |
| `GroupBlock`   | A group's operator select, its children, and *+ Condition / + Sub-group*. | [React guide](/docs/frameworks/react) |
| `useDragState` | Tracks the dragged node and computes the corrected drop index.      | [Drag and drop](/docs/guides/drag-and-drop) |

- **Operators are derived from the schema** — `ConditionRow` filters `schema.conditionOperators`
  by the selected field's `supportedOperators`, so the UI can't offer an invalid operator.
- **The value input follows arity** — it reads the operator's `arity` to decide between a text
  input and *(no value)*.
- **Reordering is just `moveNode`** — the only custom logic is the same-group
  [index adjustment](/docs/guides/drag-and-drop).

## Run it locally

```sh
git clone https://github.com/GRNRB/query-builder
cd query-builder
pnpm install
pnpm --filter './packages/*' run build   # build the libraries the example depends on
pnpm --filter basic dev                   # start the Vite playground
```
